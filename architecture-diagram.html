הסוכנים — מי עושה מה

שני סוכנים שעובדים סדרתית (לא במקביל). קופי קודם, עיצוב אחר כך. ככה אין בעיות אינטגרציה.

AGI-Copy
סוכן כתיבה שיווקית
  • 60+ נוסחאות כותרות — Dream Outcome, Curiosity Gap, מספרים ספציפיים, ועוד
  • PAS Framework — כאב, הגברה, פתרון בכל סקשן
  • פסיכולוגיה ישראלית — דוגרי, ספקנות, ספציפיות, Loss Aversion
  • אנטי-AI — 13 ביטויים חסומים (מהפכני, חדשני, פתרון מושלם...)
  • 10+ בדיקות אוטומטיות — עברית טבעית, מספרים, מגדר, CTA, מבנה
  • 14 סקשנים — Hero, Pain, Emotional Break, Cards, Process, Results, Testimonials, About, FAQ, CTA, Form, Footer
קלט: בריף JSON ← פלט: קופי JSON
AGI-Design
סוכן עיצוב ופיתוח
  • Layout מותאם לתחום — סכמת צבעים, טיפוגרפיה, ספייסינג
  • Responsive — Mobile-first, breakpoints, viewport meta
  • RTL מושלם — עברית, direction, unicode-bidi, פונט מקצועי
  • HTML + CSS מוכן לפרודקשן — קוד נקי, ביצועים, נגישות
  • 14 סקשנים ויזואליים — hero, cards, carousel, FAQ accordion, forms
  • זברה עיצובית — סקשנים כהה/בהיר לחוויית קריאה מקצועית
קלט: בריף + קופי ← פלט: HTML מלא

למה סדרתי ולא מקבילי?

מקבילי (הדיאגרמה הישנה) סדרתי (הארכיטקטורה הנכונה)
העיצוב נבנה בלי לדעת אורך הקופי העיצוב מקבל קופי מוכן ובונה סביבו
הקופי נכתב בלי לדעת מגבלות עיצוב הקופי נכתב חופשי, העיצוב מתאים
צריך שלב אינטגרציה נפרד — סיוט אין אינטגרציה — הקופי כבר בתוך ה-HTML
כותרת 3 שורות + מקום לשורה אחת = שבור ה-layout נבנה לפי התוכן האמיתי
יותר מסובך, יותר נקודות כשל פשוט, אמין, צפוי

לולאת עריכה — מה שהופך את זה לבילדר

אחרי שהדף נוצר, הלקוח יכול לבקש שינויים דרך הפלאגין. כל עריכה = קריאה ממוקדת, לא יוצרים את כל הדף מחדש.

לולאת עריכה — שינויים ממוקדים
הלקוח מבקש שינוי דרך הפלאגין — מקבל HTML מעודכן תוך שניות
"תחליף את הכותרת"
AGI-Copy בלבד — כותב כותרת חדשה, מזריק ל-HTML קיים
"תשנה צבעים לכחול"
AGI-Design בלבד — CSS edit, לא נוגע בקופי
"תוסיף חלק של המלצות"
AGI-Copy (כותב) + AGI-Design (מעצב) — מוסיף סקשן ל-HTML
"תשנה את הטון לנשי יותר"
AGI-Copy מלא — כל הקופי נכתב מחדש עם טון חדש

חוזי נתונים — מה עובר בין כל שלב

כל שלב יודע בדיוק מה הוא מקבל ומה הוא מחזיר. אין הפתעות.

מעבר מה עובר פורמט
פלאגין ← API שאלון לקוח + סוג בקשה (חדש/עריכה) POST JSON
API ← AGI-Copy בריף מובנה (תחום, קהל, כאבים, שירותים, טון) Structured JSON
AGI-Copy ← AGI-Design בריף + קופי מוכן לכל 14 סקשנים Structured JSON
AGI-Design ← QA HTML מלא (1,000-1,500 שורות) Raw HTML
API ← פלאגין HTML מוכן + metadata (כותרת, slug, תיאור) JSON { html, meta }
פלאגין ← וורדפרס יצירת/עדכון דף WP REST API
