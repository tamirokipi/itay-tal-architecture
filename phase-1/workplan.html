מה אני צריך ממך

טמפלייט אחד של דף נחיתה מלא, גנרי (לא ספציפי לנישה). בכל מקום שצריך קופי, אתה שם טקסט בסוגריים מסולסלים: {כמו זה}. זהו כל מה שאתה צריך לעשות.

אתה שולט על המבנה לגמרי. בדיוק כמו שבנית את הטמפלייט של ההירו שעבדנו עליו ביחד, אתה מחליט מה יש בכל סקשן, כמה אלמנטים, איזה סדר, איזה עיצוב. אתה רוצה 4 כרטיסים במקום 3? תשים 4. רוצה 3 שאלות במקום 5? תשים 3. רוצה להוסיף אלמנט שלא כתוב פה? תוסיף. הדף הזה שלך. אני רק ממלא את מה שבתוך הסוגריים.

CSS ID, רגע מה זה?

איתי, זה אופציונלי לגמרי, אבל אני רוצה להסביר לך מה זה ולמה זה יכול להיות שווה.

מה זה CSS ID

כל אלמנט באלמנטור יש לו שדה שנקרא CSS ID. אתה מגיע אליו ככה: לוחצים על האלמנט ← לשונית Advanced ← שדה CSS ID. אתה פשוט כותב שם שם קצר, למשל hero-h1 או pain-body.

ברגע שאתה שם CSS ID, הוא נשמר ב-JSON של הטמפלייט. כשהמנוע שלי מקבל את ה-JSON, הוא יכול לזהות כל אלמנט לפי ה-ID הזה ולדעת בדיוק מה התפקיד שלו בדף.

בלי CSS ID, מה קורה?

המנוע שלי עדיין עובד. הוא מזהה את האלמנטים לפי שמות ה-placeholders. אם כתבת {כותרת כאב}, הוא מבין שזו כותרת של סקשן כאב. אז הכל עובד גם בלי CSS ID.

אז למה בכלל CSS ID?

כי זה נותן לי שליטה מדויקת יותר. דוגמא: אם יש לך שני אלמנטים עם placeholder דומה, ה-CSS ID עוזר לי להבדיל ביניהם. זה גם עוזר לי להבין את ההקשר של כל אלמנט, באיזה סקשן הוא יושב, ולכתוב קופי שמותאם בדיוק למיקום שלו בדף.

הפורמט פשוט: section-role. למשל:

  • hero-h1 = כותרת ראשית בהירו
  • pain-body = גוף טקסט בסקשן כאב
  • cards-card2-title = כותרת של כרטיס 2
  • faq-q3 = שאלה 3 ב-FAQ

השורה התחתונה

תחליט אתה. אם אתה רוצה להשתמש ב-CSS IDs, אני אבנה את המנוע שיתחשב בזה ויתן קופי מדויק יותר. אם אתה מעדיף לא להתעסק עם זה, תגיד לי ואני אוריד את זה לגמרי מהוורקפלואו כדי לחסוך מורכבות. ככה או ככה זה עובד.

כמה כללים

  • סוגריים = קופי שלי. טקסט בתוך {} נמלא ע"י המנוע. טקסט בלי סוגריים = קבוע, אני לא נוגע.
  • Placeholder אחד גדול עדיף. במקום במיוחד ל{קהל} שרוצים {תוצאה} עדיף {badge text}. ככה המנוע שולט על מגדר ומבנה המשפט. ראית את זה בטמפלייט ההירו, אותו רעיון.
  • אתה שולט על הכמות. רוצה 4 כרטיסים? תשים 4 placeholders. רוצה 3 שאלות? תשים 3. אין מספר קבוע.
  • בלי תמונות AI. תמונות = של הלקוח. תשאיר מקום ריק עם placeholder לתמונה, הלקוח יעלה.
  • עיצוב זברה. סקשנים מתחלפים כהה/בהיר.
  • טקסט קבוע = ניטרלי. אם יש טקסט קבוע ליד placeholder, תוודא שהוא לא מכיל פעלים עם מגדר ספציפי. ככה הקופי שלי יכול להתאים לכל קהל.
